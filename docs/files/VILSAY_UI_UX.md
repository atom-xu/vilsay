## 一、设计原则

```
① 极简：用户注意力在说话，不在界面
② 原生：遵循 Apple HIG，macOS 一等公民感
③ 无打扰：AI3 后台运行，用户只感知结果
④ 可信：状态清晰，用户知道现在在做什么
⑤ 参考现有：已有前端设计作为基础，不从零发明
```

> ⚠️ 开发前必须先获取现有前端截图/设计文件，
> 在现有设计基础上调整，不重新设计。

## 三、悬浮状态条（Pill，当前实现）

> **与早期草案差异**：浮层 **不是** 60×60 常驻圆形按钮，而是 **横向品牌色胶囊条**，且 **仅在需要反馈时出现**；日常待机 **不** 显示该面板。代码：`FloatingButtonView` + `FloatingButtonController`（`NSPanel`）。

### 3.1 显示/隐藏规则（FloatingButtonController）

```
显示条件（任一成立即 orderFront）：
  · AppStatus：recording / processing / injecting / editMode
  · 或：润色完成后的短时预览条（Toast，在 floatingPreviewDismissAt 到期前）

隐藏条件：上述均不成立时 orderOut（面板不在屏幕上绘制待机态）
```

### 3.2 窗口与交互

```
容器：NSPanel（borderless、nonactivating、可跨 Space）
默认位置：主屏可见区域水平居中、靠近底边（约距底 60pt），可拖动
尺寸：内容区约宽 300pt、高 72pt（含投影留白），可随实现微调
记忆：setFrameAutosaveName("VilsayFloatingPill") 保存用户拖动位置
阴影：面板 hasShadow 关闭，阴影由 SwiftUI 胶囊形状自带
```

### 3.3 视觉规格（FloatingButtonView）

```
形态：完整胶囊（Capsule）条
背景：深色暖棕渐变条（floatBarStart → floatBarEnd，见 DesignTokens）
描边：录音/处理/Toast 时按状态使用品牌色半透明描边
录音（普通）：
  · 左侧脉冲圆点（品牌橙）+ 文案「VILSAY」+ 多段竖条波形（橙→粉→紫渐变）
录音（改词模式）：
  · 脉冲点与波形为靛蓝→紫系，文案为「听指令…」
处理中：
  · 小号 ProgressView +「思考中…」，描边偏紫
完成预览（Toast）：
  · 成功图标 + 润色结果单行截断预览 +「有误」反馈（或「已记录」）
  · Hover 可延长停留（extendFloatingPreviewOnHover）
```

### 3.4 触发方式（用户在设置中选择）

```
说明：开始/结束录音由主热键、菜单栏「开始/停止录音」等入口驱动；浮层仅在录音/处理/完成预览阶段提供反馈。

Push 模式（默认）：
  按住热键/触发 → 开始录音 → 浮层出现录音条
  松开 → 结束录音并进入处理 → 浮层切换为处理条

Toggle 模式：
  单击开始录音 → 再单击结束并开始处理
  录音中按 ESC → 取消（见 3.5）

悬浮条右侧「×」与 ESC 等价含义以设置页文案为准。
```

### 3.5 取消机制

```
录音中取消：
  ① 按 ESC（与设置中说明一致）
  ② 点击浮条右侧圆形「×」按钮（help：取消录音（与 ESC 相同））

取消后：浮层随状态结束而隐藏；无输出；不写入 raw_log（以 Pipeline 行为为准）
```

### 3.6 改词模式（视觉与文案）

```
进入改词模式后：浮条为靛蓝/紫色波形 +「听指令…」，与普通录音的橙粉紫体系区分。
用户说指令 → AI 修改选中文字；结束后浮层随流程结束隐藏。

改词指令示例（产品教育可用）：
  「改短一点」「改正式一点」「改口语一点」「翻译成英文」「纠正错别字」
```

## 四、状态系统

### 4.1 菜单栏图标状态

```
待机：麦克风轮廓，灰色
录音中：麦克风实心，红色
处理中：旋转指示器
错误：感叹号，橙色
有推荐词：待机图标 + 红色角标
```

### 4.2 状态转换

```
待机 → [触发] → 录音中
录音中 → [结束] → 处理中
录音中 → [ESC/取消] → 待机
处理中 → [完成] → 待机
处理中 → [出错] → 错误
错误 → [重试/点击] → 待机
任意状态 + AI3推荐 → 当前状态 + 角标
```

## 七、文案规范

### 7.1 通用原则

```
① 中文，不用英文（专有名词除外）
② 说人话，不说产品话
③ 错误信息告诉用户怎么做
④ 不暴露 AI3 的存在
```

### 7.2 关键文案

| 场景 | 文案 | 禁止 |
|------|------|------|
| 录音中 | 正在录音... | Recording |
| 处理中 | 正在整理... | Processing |
| 已取消 | （无提示，静默）| 已取消 |
| 改词模式 | 说指令来修改选中文字 | Edit Mode |
| 网络错误 | 网络连接失败，已切换本地模式 | Network Error |
| 权限缺失 | 需要授权才能使用此功能 | Permission Denied |
| AI3 推荐 | 发现新词，点击查看 | AI 建议 |
| 占位功能 | 即将推出 | Coming Soon |
| 免费额度用完 | 本月免费次数已用完，升级继续使用 | Quota exceeded |

### 7.3 不暴露 AI3 的说法

```
❌ 「AI3 正在分析你的语音习惯」
❌ 「用户画像更新完成」
✅ 「Vilsay 越来越了解你的表达方式」
✅ 「发现你常用的词汇，是否加入词典？」
```

## 测试流程

### 第一步：确认任务状态
打开 VILSAY_DEV_TASKS.md，找到 Cursor 标注为 🔄 或刚完成的 Task。

### 第二步：逐项验收
按照任务书中每个 Task 的「完成验证」清单逐项测试。

### 第三步：记录结果
```
通过：在 VILSAY_DEV_TASKS.md 对应行标注 ✅ + 日期
失败：标注 ❌ + 具体失败原因 + 复现步骤
阻塞：标注说明，反馈给 Cursor 修复
```

### 第四步：里程碑确认
每个 Week 结束时，确认里程碑验收节点全部通过后再进入下一 Week。

## 各阶段测试重点

### Week 2 · 前端界面测试

```
测试要点：
□ 每个界面能否正常打开和关闭
□ 界面内容是否符合 UI/UX 文档规范
□ 占位组件（P2功能）是否灰色不可点
□ Hover 占位组件是否显示「即将推出」
□ 文案是否符合 UI/UX 第六章文案规范
□ 悬浮按钮是否可以拖动
□ 悬浮按钮五种状态切换是否正确

交付物：各界面截图，与 UI/UX 文档对照确认
```

### Week 3 · 主链路功能测试

```
核心验收（里程碑）：
□ 按住热键说话，松开后文字出现在微信输入框
□ 说「嗯就是你知道吗」→ 填充词被删除
□ 选中一段文字，按热键说「改短一点」→ 文字被修改
□ 录音中按 ESC → 无任何文字输出
□ 延迟测试：10次平均 < 1500ms

测试场景：
□ 微信输入框注入
□ Safari 文本框注入
□ VS Code 注入
□ 备忘录注入
□ Slack 注入

异常场景：
□ 断网时：自动切换本地模式，菜单栏有提示
□ 权限被拒：有引导提示
□ 说空话（只有停顿）：无输出，无崩溃
□ 极长句子（> 200字）：正常处理
□ 快速连续触发热键：不崩溃
□ Push/Toggle 模式切换：立即生效
```

### Week 4 · 账号体系测试

```
□ 邮箱注册：收到验证邮件，点击后账号激活
□ 邮箱登录：正确的密码登录成功
□ 邮箱登录：错误密码有提示
□ Apple ID 登录：弹出系统授权界面，登录成功
□ Google 登录：跳转授权，回调后登录成功
□ 微信登录：（如已上线）跳转微信，回调登录成功
□ 退出登录：状态正确清除
□ 用量统计：说10句，用量 +10
□ 用量显示：设置页显示正确数字
□ 超额拦截：超过免费额度后有提示
□ 忘记密码：收到重置邮件
```

### Week 5 · AI3 测试

```
□ 说20句话后：SQLite raw_log 有20条记录
□ 取消的对话：raw_log 没有写入
□ 触发计数：第20条写入后 AI3 自动触发
□ 分析完成后：user_profile 表有数据
□ Profile 质量：人工检查内容是否合理（反馈给架构师）
□ 候选词：dictionary_candidates 有推荐
□ 词典页角标：有 pending 候选词时角标显示
□ 加入词典：点加入后在「我的词典」Tab 出现
□ 忽略词典：点忽略后不再显示
□ 清除数据：清除后 raw_log/profile/candidates 为空
□ 清除后词典：手动添加的词典条目仍然存在
```

### Week 6 · 性能与边界测试

```
性能指标（死亡线）：
□ 延迟：10次平均 < 1500ms，无一次超过 3000ms
□ 内存：待机状态 < 100MB
□ CPU：录音时 < 20%

边界场景完整测试：
□ 断网：完整流程，自动降级本地
□ 权限拒绝：麦克风/Accessibility 被拒的处理
□ 说空话：只有停顿，无输出
□ 极长句子：> 200字
□ 纯英文输入
□ 中英混合输入
□ 不支持注入的 App（终端）：优雅降级
□ 快速连续触发：不崩溃不重叠
□ Push/Toggle 模式切换
□ 改词后再改词
□ 未登录状态触发：提示登录
□ FN/Globe 键（如已实现）
```

## Bug 记录格式

发现问题时在 VILSAY_DEV_TASKS.md「未知风险登记表」填写：

```
发现日期：2026-XX-XX
问题描述：[具体描述什么情况下出现什么问题]
复现步骤：
  1. 做了什么
  2. 做了什么
  3. 出现了什么
预期结果：[应该发生什么]
实际结果：[实际发生了什么]
影响范围：[哪个功能受影响]
严重程度：P0崩溃 / P1功能不可用 / P2体验问题 / P3样式问题
```

## 文案验证清单

测试界面时对照 UI/UX 文档第六章检查：

```
□ 不出现英文（专有名词除外）
□ 不出现「Recording」「Processing」「Error」等英文状态词
□ 不出现「AI3」「用户画像」「机器学习」等技术词汇
□ 错误信息是中文且告诉用户怎么做
□ 占位功能显示「即将推出」而不是「Coming Soon」
□ 超额提示语言友好，不生硬
```

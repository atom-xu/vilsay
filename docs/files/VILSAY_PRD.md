## 一、产品定位

### 1.1 一句话定义
**Vilsay 是一个让你说话比打字更高效的输入工具，而且越用越懂你。**

### 1.2 核心价值主张

```
不是：语音转文字工具（那是微信 / 苹果自带的）
不是：录音整理工具
不是：AI 写作助手

是：说话即高质量输入
    ↓
按住热键 → 说话 → 松开 → 干净准确的文字出现在光标位置
```

**四个核心承诺，缺一不可：**

```
① 准确
   现有语音输入（微信 / 苹果自带）错别字多、同音字混淆严重。
   Vilsay 用 LLM 做第二层纠偏：
   ASR 转错的字，结合上下文语义还原正确表达。
   用户说的每个字都有意义，一个都不能丢错。

② 效率
   说完就出来，不能等。
   目标：松开热键到文字出现 < 1.5 秒。
   超过 3 秒用户会放弃，这是死亡线。
   流式输出：字逐渐出现，而不是等全部生成再显示。

③ 思路（差异化核心）
   不只是整理，是理解你在想什么。
   你说「就是那个，上次说的那个方案」——
   Vilsay 知道你的说话习惯，知道你的思维方式，
   输出的不是你说的字，是你想表达的意思。
   这是 AI3 用户画像系统的核心价值，越用越准。

④ 场景
   同一个人，发微信和写邮件，语气完全不同。
   Vilsay 根据内容语义自动识别场景：
   闲聊 → 保留口语感
   指令 → 提取动作，结构化输出
   专业阐述 → 逻辑完整，术语准确
   用户不需要手动切换，系统自己判断。
```

**三层产品目标：**

```
第一层：基础稳定（准确 + 效率）
  做到不比现有工具差，用户愿意用

第二层：良好体验（场景 + 纠偏 + 改词）
  做到明显比现有工具好，用户喜欢用

第三层：离不开（AI3 用户画像）
  越用越懂你，换掉会有损失感，用户必须用
```

### 1.3 目标用户

```
主要用户：每天需要大量文字输入的 Mac 用户

① 创业者 / 老板
  痛点：需要快速安排任务、总结会议、布置工作
  场景：说一段话 → 直接变成可发送的任务描述
  核心需求：准确、结构清晰、可以直接发出去

② 产品经理 / 项目负责人
  痛点：思维快、打字慢，想法来不及记录
  场景：随时说出需求 → 变成结构化的产品描述
  核心需求：效率、不丢失关键信息

③ 专业人士（律师 / 医生 / 咨询顾问）
  痛点：需要快速讲清楚专业逻辑，术语多、精度要求高
  场景：口头阐述观点 → 专业、准确的书面表达
  核心需求：准确性、术语识别、逻辑完整

④ 开发者
  痛点：用语音描述需求给 AI 工具更快，打字打断思路
  场景：对着 Cursor / Claude 说需求 → 直接输入
  核心需求：速度、技术术语识别

⑤ 内容创作者
  痛点：想法在脑子里，打字会打断思路
  场景：边想边说 → 快速形成初稿
  核心需求：流畅、保留原始表达意图

共同特征：
  - 说的话本身有信息密度，不是闲聊
  - 对准确性的要求高于对风格的要求
  - 每天输入量大，效率提升收益显著
```

### 1.4 平台策略

```
第一阶段：macOS（当前开发目标）
第二阶段：iOS（复用 macOS 核心逻辑）
暂不支持：Windows、Android、Web 输入
```

## 四、核心用户流程

### 4.1 首次使用流程

```
安装 App
    ↓
Onboarding 引导（3步）
    ↓
Step1：授权麦克风权限
    ↓
Step2：授权 Accessibility 权限
    ↓
Step3：注册 / 登录账号
    ↓
完成 → 进入主界面
    ↓
提示：「按住悬浮按钮或 Shift+G 开始说话」

说明：Vilsay 采用纯 SaaS 模式，用户无需填写任何 API Key，
     登录账号后开箱即用。定价方案见 PRD-SUP-001。
```

### 4.2 日常使用流程（输入模式）

```
用户在任意 App，光标在输入框
    ↓
按住悬浮按钮（Push 模式）
或点击悬浮按钮（Toggle 模式）
或按住 Shift+G（热键）
    ↓
悬浮按钮变为录音状态（红色脉冲）
    ↓
用户说话（中文/英文/混合）
    ↓
松开 / 再次点击 / 松开热键
    ↓
中途想取消 → 按 ESC 或拖离取消区域
    ↓
阿里云 ASR 转写（< 0.5秒）
    ↓
Qwen V2 Prompt 润色（流式，字逐渐出现）
    ↓
文字注入光标位置
    ↓
后台：异步写入 asr_raw_log（用户感知不到）
```

### 4.3 改词流程（Select + Speak）

```
用户在任意 App 选中一段文字
    ↓
触发热键（或点击悬浮按钮）
    ↓
Vilsay 检测到有选中文字 → 进入改词模式
    ↓
用户说指令（「改短一点」「换成正式语气」「翻译成英文」）
    ↓
AI2 接收：[原文] + [指令] → 输出修改结果
    ↓
替换选中文字
```

### 4.4 AI3 进化流程（用户无感知）

```
每次对话 → 写入本地 raw_log
    ↓
累积20条 → 触发 AI3 分析（后台异步）
    ↓
更新本地 user_profile
    ↓
发现新词 → 写入 dictionary_candidates
    ↓
菜单栏 / 悬浮按钮出现角标「1」
    ↓
用户确认或忽略
    ↓
下次润色自动更准
```

## 七、数据说明（对外）

```
音频数据
录音在设备本地处理，从不上传。录音结束即销毁。

转写与润色
转写后的文字发送至阿里云进行 AI 润色处理。
阿里云不留存您的数据，处理完成后文字返回您的设备。

本地学习数据
转写文字存储在您的设备本地，不上传。
仅用于改善您个人的使用体验。
可随时在设置中清除所有本地数据。

账号数据
邮箱、登录状态、用量记录存储在 Vilsay 服务器。
我们不出售您的任何数据。
```
